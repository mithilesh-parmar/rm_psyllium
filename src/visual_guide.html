Type scale (desktop)

Sub-heading DM Sans 500, 1.1rem, lh 1.3, ls -0.01em
Body regular DM Sans 300, 14–15px, lh 1.75
Caption / label DM Mono 400, 10–11px, ls 0.16em
CTA text DM Sans 400, 11px, ls 0.08em, uppercase

Typography rules

  • → Display type never exceeds 300 weight
  • → Italic is reserved for brand tone — not for emphasis
  • → No all-caps in headlines — only mono labels
  • → Letter-spacing is tight on display (-0.02em), loose on mono (+0.16em)
  • → No system fonts (no Inter, no Roboto, no Arial)
  • → Line height is generous on body (1.75), tight on display (0.9–1.1)
  • → DM Sans 500 (medium) is the heaviest weight used anywhere
  • → On mobile, display fonts scale down with clamp — never below 2.5rem

04 — Imagery Style Direction

Three visual modes for a complete system

Mode 01 — Macro Ingredient
Extreme close-up. Texture as beauty. Grain, powder dust, seed surface.
Mode 02 — Process & Origin
Field, facility, laboratory. Documentary feel — never staged stock.
Mode 03 — Application Context
Finished products: supplements, food, animal feed. Buyer's world, not the factory.

Photography brief

  • → Natural light preferred — avoid studio flash highlights
  • → Color palette: sage, linen, off-white, warm grey — no saturated colors
  • → Desaturated treatment, slight warm grade — not cool or clinical blue
  • → Shallow depth-of-field for macro ingredients
  • → Grain/texture visible — resists compression — signals quality
  • → No smiling people pointing at screens — it's a B2B ingredient brand
  • → Geometric compositions — rule of thirds with large negative space
  • → Leave space at edges for text overlays

What not to photograph

  • ✕ Generic green wheat field / nature stock
  • ✕ Smiling farmers in over-lit scenes
  • ✕ Blue-tinted laboratory stock imagery
  • ✕ Capsules and supplement bottles as hero shots
  • ✕ Warm orange/yellow harvest clichés
  • ✕ Graphic globe / map illustrations
  • ✕ Text on photos with poor contrast
  • ✕ Low-res product photos on white background as hero

06 — Motion & Microinteraction Direction

Motion that breathes, never performs

All motion is additive. The site should work perfectly with no motion at all. Animations exist to confirm intent and guide attention — not to entertain. Duration is always short. Easing is always ease-out or custom cubic-bezier — never bouncy or spring-based.

Entrance — Staggered content reveal

Hero headline, subline, CTAs enter with opacity 0→1 + translateY(16px→0). Stagger: 120ms between items. Easing: cubic-bezier(0.25, 0, 0.1, 1). Total: ~600ms. No sliding images — only text elements animate in.

Section — Scroll-triggered reveal

Sections enter from slightly below. Y offset: 20px → 0, opacity 0→1. Duration: 500ms. Once only — elements don't re-animate on scroll back up. Works without JS via CSS @starting-style fallback.

Primary: background darkens 10%. Ghost: background fills to 6% opacity of accent color. Text button: color deepens. All have 160ms ease-out. Active state: 80ms ease-in for instant click feel. No scale transforms on CTAs.

Transparent → white background when scrolled past 80px. Transition: background-color 200ms ease-out. A 1px bottom rule appears simultaneously. No drop shadow. Logo color inverts from white to dark.

Statistics — Count-up animation

Stat numbers (40+, 25yr, etc.) count from 0 to their value when scrolled into view. Duration: 1200ms, easing: cubic-bezier(0, 0, 0.2, 1). Fires once. If number has unit (%), count number only. Runs only if prefers-reduced-motion: no-preference.

Accessibility — Reduced motion

All translateY animations, count-ups, and horizontal scrolling panels are disabled when prefers-reduced-motion: reduce is set. Only opacity transitions ≤ 200ms remain. This is non-negotiable for pharma/healthcare buyers who may have accessibility requirements.

09 — Quality & Compliance Page Structure

The page that closes high-value procurement deals

Page Section: Hero
Quality at source.

Dark hero with full-width lab imagery. Cormorant headline: "Zero compromise. Verifiable." Short editorial paragraph establishing the quality philosophy. Scroll-anchor links to: Certifications, Testing, Traceability, Documentation.

Page Section: Certification Grid
Multi-market certification

Grid of certification badges: ISO 22000, HACCP, USDA Organic, EU Organic, FSSC 22000, Halal, Kosher, Heavy Metal / Pesticide compliant. Each with scope description and issuing body. Downloadable certificate PDFs linked inline.

Page Section: Testing Protocol
Every batch, tested

A step-by-step testing protocol — raw material intake → in-process → pre-dispatch → third party verification. Each step has a parameter list (aflatoxin, microbiological, heavy metals, pesticides, purity, moisture). Visual: horizontal timeline or numbered list.

Page Section: Traceability
Farm to formulation

Origin traceability: Batch number → farm GPS location → processing date → QC sign-off → dispatch. Support for customer audit requests. Mention of ERP/batch management system. Optional: interactive batch lookup field.

Page Section: Documentation
Documentation on demand

A document request center. Buyer fills a short form (company, product, purpose), receives: COA, TDS, Allergen Declaration, Non-GMO Statement, Country of Origin, MSDS. This is a key conversion — gated PDF downloads build the CRM pipeline.

Page Section: Audit Support
Supplier audit ready

RM Psyllium welcomes supplier audits. Factory visit coordination, remote audit support, pre-qualification questionnaire completion (SQF, EcoVadis, customer-specific). Contact form for audit scheduling. Signals confidence and transparency.

This page is essential for pharma and major food manufacturer buyers who require supplier qualification documentation before raising a PO. It should be complete, credible, and downloadable — not a marketing page. Tone: matter-of-fact, precise, evidence-based. Avoid: generic "commitment to quality" language without proof.

10 — What to Avoid

The precise list of things that undermine premium positioning

✕
Brown / orange palette

Earthy tones signal low-cost agricultural exporters, not premium ingredient suppliers. Even warm tones must be muted and desaturated.

✕
Dark navy as primary

Deep navy reads as outdated corporate or government portal. If blue is used (Palette B), it must be Prussian/ink — deeper and more considered than typical navy.

✕
Generic eco-green

Bright green leaf imagery, green gradient backgrounds, or treble-clef leaf logos. The botanical positioning must come from photography and copy — not green color coding.

✕
Hero sliders / carousels

Image sliders reduce SEO indexability, add layout instability, and signal low-quality builds. One strong, static hero. No "next slide" arrows.

✕
Auto-play video heroes

Bandwidth-heavy, slow on mobile, jarring for B2B buyers opening the site on a procurement laptop. Video (if used) is always opt-in: play button, muted by default.

✕
Heavy JavaScript animations

GSAP particle effects, Lottie files, WebGL backgrounds. These crash Core Web Vitals, harm SEO, and make procurement buyers distrust the site's professionalism.

✕
Inter / Roboto / Arial

Default system/generic fonts signal zero design intent. Cormorant + DM Sans combination is distinctive, premium, and distinct from every competitor in the psyllium space.

✕
Stock smiling people

Generic handshake or "team meeting" stock imagery destroys credibility in B2B procurement. Real facility photos, real product macro shots, or abstract editorial imagery only.

✕
Cluttered homepage

Too many CTAs, too many colors, too many overlapping sections. Premium means restraint. One CTA per section. Generous whitespace. Let typography breathe.

✕
Pill-shaped buttons

Rounded pill CTAs look SaaS-startup. This is an ingredient exporter for pharma/food buyers. 2px max border-radius on all interactive elements.

✕
Unsubstantiated claims

"World's best quality", "leading supplier", "trusted worldwide" without data. Procurement buyers require evidence. Replace with: purity %, certifications, specific markets served, batch test references.

✕
Missing multilingual SEO

hreflang tags, separate URLs per language (/de/produkte/, /es/productos/), translated meta descriptions. Not just UI translation — full SEO architecture per language.
